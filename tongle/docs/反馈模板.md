## 反馈怎么处理（你提交后会进入这个闭环）

1. **分类**：bug / 体验 / 文档 / 新需求
2. **优先级**：阻塞 → P0（优先修），影响 → P1，轻微 → P2
3. **改进**：下个版本修，修了会通知你
4. **验证**：新版发布后请你确认，或作者复现验证

阻塞类反馈作者会优先响应。你的反馈直接驱动 ke 下个版本改什么——这就是它怎么变好用的。
